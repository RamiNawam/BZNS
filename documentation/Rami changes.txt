TABLES CREATED
--------------

1. profiles
   - The central table. One row per user.
   - Stores everything from the intake wizard: business name,
     type, sector, location, borough, home-based flag, age,
     immigration status, gender, languages, preferred language,
     business structure, partner/employee info, expected monthly
     revenue, startup budget, NEQ/GST status.
   - Also stores financial snapshot inputs: monthly_expenses,
     expense_categories (JSONB), price_per_unit, units_per_month.
   - intake_completed flag tracks whether user finished onboarding.
   - intake_answers (JSONB) stores raw answers for reference.
   - RLS: ON — users can only access their own row.
   - Foreign key: user_id → auth.users(id), CASCADE DELETE.

2. roadmap_steps
   - Stores the AI-generated legal roadmap per user.
   - Each row is one step: step_order, step_key, title,
     description, why_needed, estimated_cost, estimated_timeline,
     required_documents (TEXT[]), government_url, source,
     depends_on (TEXT[]).
   - status field: 'pending' | 'in_progress' | 'completed' | 'skipped'
   - completed_at timestamp + notes field for user tracking.
   - UNIQUE(profile_id, step_key) — no duplicate steps per user.
   - RLS: ON — via profile_id → profiles → auth.uid() check.
   - Foreign key: profile_id → profiles(id), CASCADE DELETE.

3. funding_matches
   - Stores the deterministic funding scorer results per user.
   - Each row is one program: program_key, program_name,
     program_type (loan/grant/tax_credit/mentorship),
     amount_description, match_score (0-100),
     eligibility_details (JSONB), summary, application_url,
     source_url.
   - is_bookmarked and is_dismissed flags for user interaction.
   - UNIQUE(profile_id, program_key) — one score per program per user.
   - RLS: ON — via profile_id → profiles → auth.uid() check.
   - Foreign key: profile_id → profiles(id), CASCADE DELETE.

4. financial_snapshots
   - Stores the full output of the hybrid tax calculator + Claude insights.
   - Inputs stored: gross_monthly_revenue, monthly_expenses, business_structure.
   - Deterministic calc outputs: annual_revenue, gst_collected,
     qst_collected, gst_qst_remittance, net_revenue,
     federal_income_tax, provincial_income_tax, qpp_contribution,
     qpip_premium, total_deductions, monthly_take_home,
     effective_take_home_rate, quarterly_installment.
   - Claude insight outputs (JSONB): suggested_expenses,
     watch_out_flags. Text: pricing_insight.
   - UNIQUE(profile_id) — one snapshot per user (upserted on recalc).
   - RLS: ON — via profile_id → profiles → auth.uid() check.
   - Foreign key: profile_id → profiles(id), CASCADE DELETE.

5. chat_messages
   - Stores full conversation history per user.
   - role: 'user' | 'assistant'
   - content: message text
   - sources (TEXT[]): KB files cited by Claude
   - context_type: which screen the message was sent from
   - session_id (UUID): groups messages into sessions
   - RLS: ON — via profile_id → profiles → auth.uid() check.
   - Foreign key: profile_id → profiles(id), CASCADE DELETE.

6. generated_documents  [STRETCH FEATURE]
   - Stores metadata for .docx files generated by the Starter Kit.
   - document_type, file_name, storage_path, download_url, metadata (JSONB).
   - Actual files live in Supabase Storage (documents bucket).
   - RLS: ON — via profile_id → profiles → auth.uid() check.
   - Foreign key: profile_id → profiles(id), CASCADE DELETE.

7. response_cache
   - Stores pre-generated Claude responses for demo day.
   - cache_key (TEXT, UNIQUE): identifier for the cached response.
   - response_data (JSONB): the full cached payload.
   - expires_at: auto-set to 24 hours after creation.
   - No RLS — system-level table, not user-specific.

----------------------------------------------------------------

FOREIGN KEY CHAIN / RELATIONSHIPS
----------------------------------

auth.users (Supabase built-in)
    └── profiles             (user_id → auth.users.id)
            ├── roadmap_steps        (profile_id → profiles.id)
            ├── funding_matches      (profile_id → profiles.id)
            ├── financial_snapshots  (profile_id → profiles.id)
            ├── chat_messages        (profile_id → profiles.id)
            └── generated_documents  (profile_id → profiles.id)

All foreign keys use ON DELETE CASCADE — deleting a user
removes all their data automatically.

----------------------------------------------------------------

ROW LEVEL SECURITY (RLS) POLICIES
-----------------------------------

Applied to: profiles, roadmap_steps, funding_matches,
            financial_snapshots, chat_messages, generated_documents.

Pattern used:
  - profiles: FOR ALL USING (auth.uid() = user_id)
  - child tables: FOR ALL USING (
        profile_id IN (
            SELECT id FROM profiles WHERE user_id = auth.uid()
        )
    )

response_cache has NO RLS — it is a server-side system table
and is never accessed directly by the client.

----------------------------------------------------------------

PERFORMANCE INDEXES CREATED (8 total)
---------------------------------------

idx_profiles_user_id          → profiles(user_id)
idx_roadmap_profile           → roadmap_steps(profile_id)
idx_roadmap_order             → roadmap_steps(profile_id, step_order)
idx_funding_profile           → funding_matches(profile_id)
idx_funding_score             → funding_matches(profile_id, match_score DESC)
idx_chat_profile              → chat_messages(profile_id, created_at)
idx_snapshot_profile          → financial_snapshots(profile_id)
idx_cache_key                 → response_cache(cache_key)

----------------------------------------------------------------

STORAGE
--------

Bucket created: "documents" (private, not public)

Access policy:
  - Users can only access files inside their own folder.
  - Folder structure enforced: /user_id/filename.docx
  - Policy: bucket_id = 'documents' AND
            (storage.foldername(name))[1] = auth.uid()::text
